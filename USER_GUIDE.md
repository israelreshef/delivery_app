# 🚴 מדריך למשתמש - אפליקציית שליחים

מדריך זה יעזור לך להתחיל לעבוד עם מערכת השליחויות החדשה.

## 🚀 הפעלה ראשונית

### 1. הפעלת השרת (Backend)
פתח טרמינל בתיקיית `backend` והרץ:
```bash
python app.py
```
השרת ירוץ בכתובת: `http://localhost:5000`

### 2. הפעלת האפליקציה (Frontend)
פתח טרמינל בתיקיית `frontend` והרץ:
```bash
npm run dev
```
האפליקציה תרוץ בכתובת: `http://localhost:3000`

### 3. אפשרות מומלצת: הפעלה עם Docker 🐳
במקום להריץ כל שירות בנפרד, ניתן להרים את כל המערכת בפקודה אחת.
פתח טרמינל **בתיקייה הראשית** (`delivery_app`) והרץ:

```bash
docker-compose up --build
```
פקודה זו תפעיל:
*   Backend (Port 5000)
*   Frontend (Port 3000)
*   PostgreSQL Database
*   Redis (עבור Socket.IO)

## 📱 מסכי האפליקציה

### 1. דשבורד ראשי (`/courier/dashboard`)
זהו המסך הראשי של השליח.
- **סטטוס זמינות:** בראש המסך יש כפתור "התחל/עצור משמרת". וודא שהוא דלוק כדי לקבל משלוחים.
- **מפה:** מציגה את המיקום שלך ומיקומי איסוף/מסירה בזמן אמת.
- **לשוניות:**
    - **מפה:** תצוגה ויזואלית של כל המשלוחים הזמינים באזור.
    - **פעיל:** פרטי המשלוח הנוכחי שלך (כתובות, כפתורי Waze, עדכון סטטוס).
    - **דוחות:** סיכום יומי מהיר.

### 2. דוחות ורווחים (`/courier/stats`)
צפייה בביצועים שלך:
- גרף הכנסות שבועי.
- היסטוריית משלוחים מלאה.
- סיכום טיפים ודירוג.

### 3. תהליך משלוח
1.  **קבלת משלוח:** כרטיס לבן יקפוץ על המפה כשתיכנס הזמנה חדשה. לחץ **"קבל"**.
2.  **איסוף:** נווט לחנות (Waze), אסוף את החבילה ולחץ **"נאסף"**.
3.  **מסירה:** נווט ללקוח, מסור את החבילה ולחץ **"נמסר"**.
4.  **סיום:** המשלוח יעבור להיסטוריה והרווח יתעדכן.

`Socket Connected: <ID>`
אם לא, וודא שהשרת (backend) רץ.
